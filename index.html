종합은 요약이 아닙니다

요약은 길이를 줄이고, 종합은 판단 가능한 구조를 만듭니다.

Cognitive load is a product metric

좋은 AI 제품은 더 많이 말하는 도구가 아니라 사람이 덜 소모되게 하는 시스템입니다.

Context precedes wisdom

데이터는 정보로, 정보는 지식으로, 지식은 실행 가능한 지혜로 올라가야 합니다.

Operations are part of the product

기업 AI는 권한, 감사, 비용, 보안 위에서 실제로 굴러가야 합니다.

DIKW 위계는 맥락과 지혜를 잇는 언어입니다

맥락은 데이터를 정보와 지식으로 올리는 조건이고, 지혜는 그 조건 위에서 선택하고 실행하는 능력입니다.

Data

아직 해석되지 않은 흔적

문서, 메일, 로그, VOC, 일정처럼 각각 흩어진 원자료입니다.

Information

맥락이 붙은 사실

누가, 언제, 어떤 기준으로 남긴 것인지 알 수 있을 때 판단 재료가 됩니다.

Knowledge

관계로 설명되는 구조

사실 사이의 원인, 패턴, 예외, 리스크가 연결될 때 이해 가능한 구조가 됩니다.

Wisdom

선택과 실행의 기준

충분히 연결된 맥락 위에서 무엇을 선택하고 실행할지 정하는 단계입니다.

로고는 맥락의 상승을 그립니다

겹친 블록은 흩어진 자료를, 상승하는 곡선은 DIKW 위계를 따라 판단 가능성이 높아지는 흐름을 뜻합니다. 마지막 금색 점은 사람이 선택하고 실행할 수 있는 순간을 가리킵니다.

Layered context
문서, 메일, 로그, VOC처럼 분산된 단서가 하나의 구조로 겹쳐집니다.
Rising path
데이터는 맥락을 얻어 정보와 지식이 되고, 지혜는 그 위에서 선택을 가능하게 합니다.
Color system
짙은 남색은 판단의 기준, 틸은 맥락화, 금색은 선택과 실행을 나타냅니다.

오늘의 초점

LLM 시스템이 분산된 기업 자료를 어떻게 낮은 인지 부하의 의사결정으로 바꿀 수 있는지 탐구합니다.

  • 01 Context systems

    관계, 출처, 기준, 리스크를 함께 보존하는 지식 구조.

  • 02 Decision interfaces

    오늘 결정할 것과 확인할 가정을 드러내는 화면.

  • 03 Enterprise AI rails

    인증, 권한, 보안, 감사, 사용량 책임이 작동하는 운영 기반.

  • 04 Agentic workflows

    반복 탐색은 줄이고 근거 확인과 사람의 판단은 남기는 흐름.
